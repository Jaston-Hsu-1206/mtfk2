# 業務規則：產品點購合規檢驗 (Order Rules)

## 1. 基礎實作規劃 (人類好閱讀版)

本規則旨在維護天下茶屋對品質的專業堅持，規範系統如何攔截不當的點餐組合：

### 系統攔截邏輯

- **熱飲溫度檢核**：當顧客選擇的飲品溫度是「熱 (Hot)」時，則系統應自動鎖定並禁止加入「珍珠 (Boba)」。
- **系統提示語**：當攔截發生時，系統應顯示提示訊號如下：「很抱歉，基於茶飲品質堅持，熱飲暫不提供珍珠加購，建議您改選冰飲。」

---

## 2. 進階實作規格 (AI好理解版)

### 品項組合校驗表 (Validation Matrix)

| 點餐條件 (Input) | 系統權限 (Status) | 系統反饋 (Message) |
| :--- | :--- | :--- |
| **溫度 == "熱 (Hot)"** | 禁止珍珠 (Denied) | 提示：熱飲暫不提供珍珠加購 |
| **溫度 == "去冰/微冰/全冰"** | 允許珍珠 (Allowed) | 無警告，正常顯示結帳按鈕 |

# 業務規則：氣候感測聯動推薦 (Weather Recommendation)

## 1. 基礎實作規劃 (商業邏輯)

當系統透過 `get_weather_skill` 取得即時氣溫後，應執行以下自動化動作：

- **高溫判定**：若氣溫 **高於 28°C**，系統偵測為「炎熱環境」，主推冷飲：「特調奶茶（冰）」。
- **常溫判定**：若氣溫落在 **15°C ~ 28°C**，系統偵測為「舒適環境」，主推招牌茶類：「天下紅茶」或「天下綠茶」。
- **低溫判定**：若氣溫 **低於 15°C**，系統偵測為「寒冷環境」，主推熱飲：「天下紅茶（熱）」。

---

## 2. 進階實作規格 (AI好理解版)

### 推薦決策表 (Recommendation Matrix)

| 觸發條件 (Condition) | 氣溫區間 | 推薦產品 (Primary) |
| :--- | :--- | :--- |
| **高溫** | `current_temp > 28` | 特調奶茶（冰） |
| **常溫** | `15 <= current_temp <= 28` | 天下紅茶 / 天下綠茶 |
| **低溫** | `current_temp < 15` | 天下紅茶（熱） |
